你問的兩個點都很準：setTodos( ... ) 的括號到底是什麼？裡面那個 () => { } 又是什麼？
=======================================================================

你貼的程式碼（我先原樣放在這裡方便對照）：

setTodos(
  todos.map((todo) => {
    if (todo.id === id) {
      return { ...todo, isCompleted: !todo.isCompleted };
    }
    return todo;
  }),
);

------------------------------------------------------------
1) setTodos( ... ) 的第一層括號 ( ) 是什麼？
------------------------------------------------------------

答案：
- setTodos( ... ) 的 ( ) 不是「contents」的意思
- 它是「呼叫 setTodos 這個函式」時，放進去的 input（輸入參數）

也就是：
- setTodos 的 input = 你要更新成什麼「新 state」

在這裡：
- 你把 input 設為：todos.map(...) 的結果（也就是“新的陣列”）

所以更直覺一句話：
- setTodos(新todos陣列)

------------------------------------------------------------
2) 那 setTodos 的 input 裡面為什麼又有一個函式 () => { }？
------------------------------------------------------------

因為：
- todos.map(...) 需要你提供一個「callback 函式」
- 這個 callback 會被 map 對每一個 todo 呼叫一次

也就是：

todos.map( callback )

這裡 callback 就是：
(todo) => { ... }

所以你看到的「第二個函式」不是給 setTodos 的
而是給 map 的。

結構其實是這樣：

setTodos(
  todos.map( callback_for_map )
)

------------------------------------------------------------
3) 整段其實可以分成 3 層
------------------------------------------------------------

第 1 層：呼叫 setTodos（更新 state）
- setTodos(  ???  )

第 2 層：計算新的 todos（用 map）
- ??? = todos.map(  ???  )

第 3 層：map 的 callback（對每個 todo 怎麼處理）
- ??? = (todo) => { ...return... }

------------------------------------------------------------
4) 逐層用「input / output」看會最清楚
------------------------------------------------------------

(1) map 的 input / output
--------------------------------
input：
- 原本的 todos 陣列
- callback 函式： (todo) => { ... }

output：
- 一個新的陣列（長度一樣）
- 裡面每個元素是 callback 回傳的結果

(2) setTodos 的 input / output
--------------------------------
input：
- “新的 todos 陣列”（map 的 output）

output：
- setTodos 本身沒有你要用的回傳值（你不需要接它回傳什麼）
- 它的效果是：更新 state → React 重新 render

------------------------------------------------------------
5) 所以你問：「裡面是不是又創了一個函式？」——是的，但它是給 map 用的
------------------------------------------------------------

你這句問得很好：
“inside the input, we created another function in this case is the ()=>{} function?”

答案：
- 對，你在 map(...) 裡「定義了一個函式」(todo) => { ... }
- 但它的用途是：
  讓 map 對每個 todo 跑一次，決定要回傳哪個 todo

它不是 setTodos 的函式內容，而是 map 的 callback。

------------------------------------------------------------
6) 你可以用一個更像英文的句子理解它
------------------------------------------------------------

“Set todos to the result of mapping over todos with this rule.”

翻成結構就是：
- setTodos(  (todos map with rule)  )

------------------------------------------------------------
7) 小補充：setTodos 其實也有另一種「input 也是函式」的寫法
------------------------------------------------------------

你現在是：
- setTodos(新陣列)

另外一種常見的是：
- setTodos((prevTodos) => 新陣列)

這種情況下：
- setTodos 的 input 真的就是一個函式

例如：
setTodos((prevTodos) =>
  prevTodos.map((todo) => ...)
);

這裡才是「setTodos 的 input 是 function」。

------------------------------------------------------------
8) 最短結論（直接回答你的問題）
------------------------------------------------------------

- setTodos( ... ) 的第一層 ( ) 是「呼叫 setTodos 時的 input（要更新成的新 state）」。
- 你放進去的 input 是 todos.map(...) 的結果。
- map(...) 裡的 (todo) => { ... } 是「map 的 callback 函式」，不是 setTodos 的內容。

如果你想，我也可以用同樣 txt 格式把 “setTodos(新值)” vs “setTodos(prev => 新值)” 兩種情況做一個對照表，這會讓你更不混亂。
